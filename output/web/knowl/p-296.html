Output redirection with > or >> is a feature in Linux that allows to change the standard input/output devices. The standard input device is the keyboard, and the standard output device is the screen, but sometimes we want to change this. With output redirection, the above standard output can be changed to go into a file, and that is exactly what we did above. Note that the > is the output redirection operator used for overwriting files that might or might not already exist in the directory, while the >> is an output operator that appends the data of an existing file (or creates a new file by that name if one did not already exist.) So, > should be used judiciously.
